# 038 Goal Timeline + Resume

## 决策
M38 在 M37 驾驶舱基础上增加长任务可观测性与恢复能力。

### 1. 桌面恢复策略
- **桌面重启后只发现未完成任务，不自动继续执行**
- 用户必须点击"恢复任务"才调用 runAgentLoop
- 原因：避免无人类监督下自动执行危险操作

### 2. pending_permission 行为
- 显示"等待人工批准"，状态为"已暂停"
- **不自动批准**，必须人工决策
- 原因：permission gate 是安全边界

### 3. max_steps 行为
- 达到最大步数后显示"已停止"+"已达到最大步数"
- 状态为 stopped，恢复按钮不出现
- 原因：max_steps 是用户设定的预算上限

### 4. failed 行为
- 显示"失败"+ 错误信息
- 显示下一步建议
- 原因：用户需要知道为什么失败和如何修复

### 5. 时间线展示
- 空: "暂无长任务记录"
- 有事件: 显示最近5条，每条显示事件类型/步数/状态
- 不显示原始 JSON 大块

### 6. 证据展示
- 空: "暂无证据"
- 有记录: 显示数量和最近摘要

### 7. 不新增危险能力
- M38 只增加薄的 client 方法、UI 组件和测试
- 不改 permission gate / tool executor / planner 核心语义
