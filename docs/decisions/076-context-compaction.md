# M76 Context Compaction — 设计决策

## 决策背景
Biot 现在有 5 个记忆层（M71-M75），但各层独立查询，无法一次获得完整上下文画像。M76 将这些记忆层组合为结构化压缩摘要，支持 token budget 控制和 Markdown/JSON 双输出。

## 决策 1：组合而非生成
**选择**：ContextCompactionService 读取各记忆层服务，组合成结构化摘要。不调用 LLM。

**理由**：
- 文档明确："不调用 LLM 也能工作"
- 各记忆层已有确定性的结构化输出，无需额外推理
- 组合速度快、无 API 成本、可复现

## 决策 2：安全硬规则永不截断
**选择**：`active_constraints` 包含所有安全硬规则，不受 max_items 限制。

**理由**：
- 文档明确："压缩不能丢安全硬规则"
- 安全规则是上下文的基础，截断它们意味着新窗口可能在无防护状态下工作
- 实现中硬编码了关键安全规则的补充检查

## 决策 3：Markdown 输出优先中文
**选择**：`to_markdown()` 输出中文 Markdown，适合直接复制给新窗口 AI。

**理由**：
- 所有用户可见 UI 必须中文
- Markdown 格式适合 AI 窗口的上下文注入
- JSON 格式保留给程序化消费

## 风险
- Token 估算为粗略值（中文 ~0.7 token/字），实际以模型 tokenizer 为准
- 摘要精度依赖各记忆层数据质量
