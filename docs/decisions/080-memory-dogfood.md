# M80 Memory Dogfood — 设计决策

## 决策背景
V3 记忆层（M71-M79）已完成 9 个组件的开发。M80 是大复盘门，需验证所有组件是否协同工作、是否满足"新窗口更稳接手项目"的产品目标。

## 决策 1：独立就绪度服务，不耦合到各组件
**选择**：MemoryDogfoodService 独立发起对各组件的检查，各组件不内置 dogfood 逻辑。

**理由**：
- 狗粮检查是阶段性审计，不应污染各组件逻辑
- 独立服务可以按需扩展检查项
- 失败时不影响各组件正常运行

## 决策 2：12 项检查覆盖功能 + 安全 + 交叉验证
**选择**：9 项单组件检查 + 3 项交叉检查（source_refs、无自动执行、未进入 M81）。

**理由**：
- 单组件检查确保每个都能独立工作
- 交叉检查确保组件间一致性（source_refs 可追溯）
- 安全交叉检查防止隐性自动执行路径

## 决策 3：M80 通过/失败明确表态
**选择**：`ready_for_next` 字段明确表示是否允许进入 M81。

**理由**：
- 大复盘门必须给出明确结论，不能含糊
- 所有单组件检查通过 + 所有交叉检查通过 = ready_for_next: true
- 任一失败 = ready_for_next: false

## 风险
- Dogfood 检查是快照评估，不保证组件在运行时不会回归
- 组件内部异常（如网络问题）可能导致 false negative
