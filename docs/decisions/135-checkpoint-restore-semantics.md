# M135 Decision: Checkpoint 恢复只做显式文件级恢复

## 决策

M135 的恢复语义采用“显式确认 + 文件级写回”的最小闭环：只恢复 checkpoint 中保存的普通工作区文件内容，不使用 git 命令做仓库级回滚。

## 原因

项目安全底线要求恢复能力可验证、可审计、可阻断。`git reset` 或隐式 rollback 的影响面过大，容易覆盖爸爸或其他 Agent 的并行改动，也会绕开已有 PermissionGate 和审计语义。

文件级 restore 的优点是：

- 行为边界清楚。
- 测试可以直接验证文件内容变化。
- 可以逐路径执行 workspace containment 检查。
- 可以对秘密路径做强阻断。

## 取舍

- 该方案不是完整版本控制系统，不能恢复未保存进 checkpoint 的文件。
- 大文件和秘密路径不会恢复，需要人工处理。
- 后续如要做 UI 恢复，需要在此 API 外再加爸爸批准界面和审计确认。

## 风险

- checkpoint 是轻量快照，不适合长期保存大量文件。
- 旧 checkpoint 如果已含敏感内容，仍需后续清理策略；M135 开始新建 checkpoint 会跳过秘密路径。
