# M79 Memory Search UI — 设计决策

## 决策背景
M71-M78 的记忆层 API 已就绪，但爸爸只能在终端调用 API，缺乏直观的搜索入口。M79 在桌面端增加只读记忆搜索面板，让爸爸能通过 UI 查决策、失败、偏好、项目画像、代码地图。

## 决策 1：纯展示组件，不持有数据
**选择**：MemorySearchPanel 通过 props 注入 API 方法，每次搜索实时调用后端 API。

**理由**：
- 不缓存记忆数据到前端，避免数据一致性问题和 secret 泄露风险
- 遵循现有面板模式（CheckpointPanel、SideChatPanel 等）
- 搜索结果实时反映最新状态

## 决策 2：Tabs 分类筛选 + 关键词搜索
**选择**：6 个 tab（全部/决策/失败/偏好/项目/代码地图）+ 文本框关键词搜索。

**理由**：
- 文档明确："tabs 或 filter：全部 / 决策 / 失败 / 偏好 / 项目 / 代码地图"
- 分类筛选减少不相关结果
- 关键词搜索支持跨分类

## 决策 3：不提供写入/删除/执行按钮
**选择**：面板只渲染搜索结果，无任何修改操作。

**理由**：
- 文档明确："只读，不提供写入、删除、执行按钮"
- 记忆系统是上下文参考层，不应从前端直接修改
- 安全边界：写入操作需通过 PermissionGate

## 风险
- 搜索性能依赖后端 API 响应速度
- 跨分类搜索需要多次 API 调用（每个 tab 一次）
