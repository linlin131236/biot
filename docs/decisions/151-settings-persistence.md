# M151 Decision - 真实设置持久化

## 决策

M151 将设置中心从静态展示升级为真实读写，核心改动为新增桌面设置持久化后端服务和安全前端接入。

## 背景

M144-M150 完成了液态玻璃设置中心的产品化展示，所有设置页面均为静态 mock 数据。主题、语言、模型配置、默认工作区等均无法在重启后恢复。M151 作为 M151-M160「真实可用桌面端 Beta」的第一项，目标是让设置中心至少具备主题、语言、默认工作区和模型提供方状态的真实读写能力。

## 方案选择

| 方案 | 描述 | 决策 |
|------|------|------|
| A | 全量 SQLite 存储设置 | ❌ 过重，设置数据量小不需要数据库 |
| B | 纯 localStorage（前端独立） | ❌ 后端模型配置无法共享，主题与模型状态分裂 |
| C | `.bolt/desktop-settings.json` + 独立 API key 文件 | ✅ 轻量、安全、与现有 `.bolt/` 目录一致 |

### C 方案细节
- 设置文件：`.bolt/desktop-settings.json`（JSON，不含明文 key）
- API key 文件：`.bolt/desktop-api-key`（文件权限 600，不进入版本控制）
- 后端提供 REST API，前端通过现有 `harnessClient` 安全调用
- 主题存储在后端以便统一管理，但前端也可通过 CSS class 即时响应
- 语言固定简体中文，但展示真实状态

## 风险

- API key 存储路径需要确保文件权限正确（600）
- 设置文件损坏时需要降级为默认值
- 前端初始化顺序：先加载设置再渲染，避免闪烁
- `.bolt/` 目录可能不存在于新安装，需要自动创建
