# Decision 141: Liquid Glass Visual System

## 决策

M141 将桌面端主界面统一到液态玻璃视觉系统，并把软件内称谓从私人化表达切换为面向公开产品的“用户/人工批准/用户确认”。

## 背景

此前桌面端已经有工作台骨架，但页面质感、主题完整度和产品化文案不足。用户明确要求界面更高级、更科技、更有液态玻璃质感，同时要求软件内不要出现私人称呼，因为新软件面向更多用户。

## 方案

- 建立 `liquidGlassShell.css` 统一视觉 token。
- 用 `biotLiquidBorder::before` 实现慢速流光玻璃边框。
- 首页保持任务输入为第一屏，不做营销式 landing page。
- 设置页保留高密度产品配置结构，避免装饰性卡片堆叠。
- 产品 UI 文案统一为中文，且使用通用用户称谓。

## 安全边界

- 本 milestone 只改视觉和文案，不新增自动执行能力。
- 写入、apply、恢复、发布仍需人工批准。
- renderer 不暴露 `ipcRenderer` / `fs` / `shell` / `process`。
- 不使用 `as any` / `unknown as`。

## 取舍

- 先做系统级 shell、首页和设置页，而不是一次性重绘所有历史面板。
- 流光边框使用 CSS 动效实现，避免引入新的渲染库和性能风险。
- 字体大小使用固定值与媒体查询，不使用 viewport-based `clamp()`，降低窗口尺寸变化造成的布局抖动。
