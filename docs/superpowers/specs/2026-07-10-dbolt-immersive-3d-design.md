# d/bolt 沉浸式 3D 官网与桌面工作台设计

**状态：** 已确认，等待规格审阅  
**日期：** 2026-07-10  
**范围：** 新建 d/bolt 品牌官网；为 Bolt 桌面端定义统一的沉浸式 3D 工作台语言。现有 `bolt-cn-site` 不在本次改动范围内。

## 目标与边界

d/bolt 是一款本地优先、权限可控、过程可审计的桌面 AI Agent。体验必须传达：Agent 能理解项目和执行任务，但每次有副作用的动作都经过用户可见的审批，并留下可追溯证据。

本设计使用一座“权限金库”作为官网与软件共享的空间隐喻。3D 负责表达状态、层级和受控边界；高频、精确、有风险的操作必须保持在固定、可访问的 2D 控件中。3D 场景不得代替现有的 PermissionGate、diff 审核或显式确认。

## 技术方向

官网和桌面端共享 React + React Three Fiber 的核心场景、材质、颜色、状态映射和动效规则。

- 官网提供受限的镜头视差与滚动编排，侧重品牌叙事。
- Electron 桌面端采用“2D 工作台覆盖 3D 金库”的混合界面，侧重任务执行与审查效率。
- 任何一个 3D 状态都必须有等价的文字、图标或 2D 面板状态；所有关键操作可通过键盘和常规表单完成。

## 共享空间隐喻：权限金库

中心是一个低频呼吸的金属核心，外层由三道可被观察的轨道包围：

1. **项目理解轨道**：工作区、文件上下文和任务计划进入核心。
2. **权限审批轨道**：工具请求、代码 diff 和风险级别穿过可见闸门；待审批或被阻断时使用红色。
3. **证据轨道**：执行记录、测试结果、检查点和失败记忆沉淀为可回看的轨迹。

核心不会承诺“完全自主”。它始终被闸门与证据环包围，以强调本地、可控和可审计。

## 新官网

首屏即为权限金库。初始视角聚焦核心，用户鼠标移动只带来有限幅度的视差与轻微倾转，不自动旋转，不强制镜头运动。

滚动以三幕展开：

1. **理解项目**：工作区和上下文以克制的空间信号进入核心。
2. **请求审批**：变更穿过权限闸门；页面给出 diff 与风险示例。
3. **留下证据**：执行轨迹凝结为检查点、测试结果与可回溯记录。

官网的文案与界面示例优先说明产品能力，场景只作叙事，不应遮挡主标题、说明、CTA 或关键可读文本。

## 桌面工作台

桌面端将金库放在工作区中央，周围是稳定的 2D 工作区：

- 左侧固定显示任务、工作区和项目上下文。
- 中央 3D 金库显示当前阶段、审批状态和执行健康度。
- 右侧固定显示审批队列、代码 diff、证据、失败恢复与详情。

点击核心或轨道仅用于定位和筛选状态。批准、拒绝、编辑、复制、提交命令和确认写入均使用固定的 2D 面板与键盘可访问控件。3D 金库是状态控制面，不是游戏化的导航替代品。

## 视觉语言

- 背景：石墨黑与近黑色，延续当前 Bolt 的工业化界面。
- 材质：钛灰磨砂金属、低反射玻璃、细线网格与受控粒子。
- 文字：冷白与灰阶，保证内容层级和可读性。
- 警示：红色只表达风险、待审批、拒绝和阻断；不用于一般装饰。
- 禁止：通用紫蓝霓虹、持续炫光、自动镜头漫游及无意义的粒子噪声。

## 性能、无障碍与失败降级

- 常规桌面硬件上以 60fps 为目标；场景更新与业务状态解耦，避免每次 UI 渲染重建 WebGL 资源。
- 使用 `prefers-reduced-motion`、触摸设备能力和帧耗时作为降级条件。
- 降级时展示等价的静态 2D 金库图、轨道状态和面板信息；不丢失产品功能或信息。
- 无 WebGL、WebGL 初始化失败、上下文丢失和资源加载失败时自动降级，并向用户给出可恢复提示。
- 所有状态使用文本和非颜色线索表达；焦点顺序、快捷键和屏幕阅读器语义不依赖 3D 画布。

## 验收标准

1. 官网首屏能在支持 WebGL 的设备上展示权限金库，并清楚表达三幕故事。
2. 官网在减少动态、无 WebGL 或性能不足时自动呈现完整静态替代体验。
3. 桌面端的任务、审批、diff 和证据可在不操作 3D 画布的情况下完整使用。
4. 3D 场景能准确反映计划、待审批、执行、验证、失败与恢复状态，不拥有任何绕过安全链路的写入能力。
5. 常规设备上的空闲和交互帧率、首屏加载、键盘路径和降级路径均有自动化或可复现的验收记录。

## 非目标

- 不修改或替换现有 PermissionGate、差异审查、证据存储与执行权限规则。
- 不把 3D 做成自由漫游游戏、虚拟现实体验或唯一交互路径。
- 不在本设计阶段重构现有 Bolt 桌面端或替换既有 `bolt-cn-site`。
