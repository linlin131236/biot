# M97 Exec Plan - 失败解释体验

## 目标
把失败分类、可能原因、建议动作做成中文解释面板，让失败不再只是一段日志。

## 参考资料
- OpenClaw实际场景学习报告：失败解释要区分用户可处理、系统可处理和需要暂停。
- Agent产品化流水线：解释面板不能自动修复。

## 实施
- 新增 `FailureExplanationPanel.tsx` 和测试。
- 新增 `failure_explanation_api.py`，输出失败类型、说明、建议动作。
- 不新增 retry/fix/apply 入口。

## 验收
- 中文解释失败原因。
- 明确显示建议但不自动执行。
- 测试覆盖有失败和空状态。
